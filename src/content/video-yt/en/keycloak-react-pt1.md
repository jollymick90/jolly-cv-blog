## The Fundamental Problem

React is designed for **Single Page Applications (SPA)**: a single HTML page that loads a JS bundle, then React handles all navigation and rendering client-side.

Keycloak, however, uses **FreeMarker Templates** — a Server-Side Rendering approach where the server generates ready-to-use HTML for each page.

---

## The Strategy: React as an Enhancement Layer

Instead of using `ReactDOM.createRoot(document.getElementById('root'))` on an empty div, we mount React on an element already present in the HTML generated by Keycloak.

```js
import { createRoot } from 'react-dom/client';
import LoginForm from './LoginForm.jsx';

const container = document.getElementById('react-login-form');
if (container) {
  createRoot(container).render(<LoginForm />);
}
```

In the `.ftl` file:
```html
<div id="react-login-form" 
     data-action="${url.loginAction}"
     data-realm="${realm.name}">
</div>
<script src="${url.resourcesPath}/js/bundle.js"></script>
```

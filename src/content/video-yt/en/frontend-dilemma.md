## Overview

This was the first live event of 2025 for the BacaroTech community, streamed on Discord. The talk covers one of the most common questions for developers at any level: **which frontend framework should I learn?**

The answer is never simple — but the reasoning process matters more than the destination.

---

## The Evolution of JavaScript

We traced the history of the frontend ecosystem:

1. **The jQuery Era** — DOM manipulation made easy. Everything was imperative.
2. **The MVC Frameworks** — Backbone, Ember, AngularJS (1.x). Structure emerged.
3. **The Component Revolution** — React introduced components as the unit of UI.
4. **The Modern Era** — Angular (2+), Vue, Svelte, and now React Server Components.

Understanding this history helps you see *why* each framework exists and what problem it was solving.

---

## CSR vs SSR vs SSG

One of the core concepts covered:

| Approach | Description | When to Use |
|----------|-------------|-------------|
| CSR | Browser downloads JS, renders in client | Dashboards, SPAs |
| SSR | Server renders HTML per request | SEO-critical apps |
| SSG | Pages pre-rendered at build time | Blogs, docs, portfolios |

Most modern frameworks (Next.js, SvelteKit, Nuxt) support all three.

---

## The Virtual DOM Explained

React's Virtual DOM is often misunderstood:

- It's not faster than the real DOM by default
- It's a **programming model** that allows React to batch and optimize updates
- Svelte eliminates it entirely by compiling to direct DOM operations

---

## How to Choose

The talk concluded with a practical decision framework:

1. **Job market** — React dominates job listings; learn it for employment
2. **Project type** — Angular for enterprise; Svelte for performance-critical apps; Vue for gradual adoption
3. **Team size** — Angular's opinionation helps large teams; React's flexibility suits small teams
4. **Your goals** — Learning? React. Speed? Svelte. Enterprise? Angular.
